Conceptos fundamentales 

1. Antes de conocer Git mantenía mis archivos en una carpeta, teniendo que entrar a cada carpeta y abrir los archivos para verificar si es la versión correcta o no, Git podría haber resuelto esto de manera muy sencilla.

3. Git es un controlador de versiones que controla y documenta todos los cambios realizados en un archivo, esto ocurre localmente, mientras que GitHub es una plataforma para subir repositorios locales, compartir con otros, reportar bugs, etc. Ambas herramientas van de la mano, Git controlando el historial del código y GitHub permitiendo la colaboración, el trabajo en equipo sin estar pisando el código de los demás y copias de seguridad. Se puede utilizar Git sin GitHub si solo quieres llevar el historial de cambios de manera local.
